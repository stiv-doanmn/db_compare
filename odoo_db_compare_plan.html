Screen flow

1
Config
user input
Nhập DSN cho Odoo 17 và Odoo 19 (host, port, dbname, user, password)
Test connection — hiển thị badge OK / FAIL per DB
Custom prefix config: nev_, astra_ — editable, có thể thêm bớt
Sau khi cả 2 DB connect OK → unlock nút Run Schema Diff
2
Schema diff — phase 1
auto
Query information_schema cả 2 DB — so sánh columns, types, nullable, defaults
Classify mỗi bảng: custom (prefix match) hoặc base
Filter tabs: All · Changed · New · Dropped · Identical
Per table: hiển thị added cols / dropped cols / type changed / nullable changed
User review xong → Proceed to Data Compare
3
Table selection — phase 2
user chọn
Hiển thị toàn bộ bảng tồn tại ở cả 2 DB kèm thông tin:
ColumnNguồn dữ liệuGhi chú

v19 rowspg_class.reltuplesEstimate, không scan toàn bảng

4
Progress — phase 3
SSE stream
Per-table row: table name · mode · status badge · progress bar · time elapsed
Status pipeline: queued → running → done / warning / error
Live counter: Done X · Warning X · Error X · Total X

5
Report
export
Summary cards: Tables OK · With warnings · Errors · Skipped
Per-table accordion: schema diff recap · row count diff · missing IDs · value mismatches
Export: Download JSON · Download HTML

Data compare logic

StepMode áp dụngLogic
COUNT checkfull, intersection, count-onlyCOUNT(*) cả 2 DB → Δ rows
Missing ID scanfull, intersectionEXCEPT query — IDs có ở v17 mà không có v19 và ngược lại
Column scope resolvefull / intersectionfull = tất cả cols · intersection = cols tồn tại ở cả 2
Value difffull, intersectionKeyset pagination theo id, batch 1000 · MD5 row hash trước, chỉ fetch detail khi hash khác

Bảng không có cột id → tự động fallback về count-only

Constraints

DB connect qua host:port, cần read-only user
Không cần auth — internal tool
Job result giữ in-memory theo job_id UUID
Custom prefix: nev_, astra_ — configurable
pg_class.reltuples cho estimate — không chính xác 100%
Rename column không detect được tự động — report là dropped + added
